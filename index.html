One-time setup

This tool reads each diagnostics center's data from its own Google Drive folder and writes checked reports back there. To talk to Google Drive from the browser, it needs your Google OAuth credentials (created once, free).

Create a Google Cloud project & OAuth Client ID

Open the Google Cloud Console → Credentials. Create project → Create Credentials → OAuth client ID → Web application. Under Authorized JavaScript origins add the exact URL this page is served from (e.g. http://localhost:8000 or your hosting URL). Copy the Client ID.

Enable the Google Drive API & Picker API

In APIs & Services → Library, enable Google Drive API and Google Picker API. Then create an API key (Create Credentials → API key) and copy it.

Paste them here (saved only in this browser)

Why sign-in every session? A browser page cannot access Google Drive without a user explicitly granting permission — this is Google's security model. You'll click “Sign in with Google” once per session. The app requests the drive.file scope, meaning it can only see and modify files it creates or that you open with it — never your whole Drive.
All processing (parsing, duplicate matching, analytics) happens in your browser. Patient data is never sent to any third-party server — only to/from your own Google Drive.
